## Overview

This portfolio documents hands-on learning across Microsoft Azure administration and Identity & Access Management (IAM) using Microsoft Entra ID, Windows authentication logs, and Role-Based Access Control (RBAC).

The objective is to build operational capability in real-world cloud environments through structured lab work focused on:

- Azure resource provisioning and governance
- Identity lifecycle and authentication behaviour
- Security log analysis and access troubleshooting

All labs are based on controlled Azure environments designed to simulate enterprise cloud operations.

## Architecture Model

This portfolio is structured across two core cloud domains:

### Azure Infrastructure (1.x)
- Resource provisioning
- Networking and segmentation
- Compute and storage
- Monitoring and observability

### Identity and Access Management (2.x)
- Authentication systems
- Access control (RBAC)
- Identity lifecycle management
- Security governance and analysis

Together they demonstrate how Azure infrastructure and identity systems interact in enterprise environments.

---

## Evidence-Based Learning Approach

All work is based on:

- Controlled Azure lab environments
- Identity and authentication system behaviour
- Log-based investigation and validation
- Simulated enterprise cloud scenarios

The goal is to understand how Azure and Entra ID behave in operational conditions, not only theoretical configuration.
